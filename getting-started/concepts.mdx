使用起源云之前，先了解几个基础概念。这些术语会在整个文档中反复出现。

## 术语速查

- **[实例](#实例)** - 一台 GPU 服务器
- **[镜像](#镜像)** - 系统环境快照
- **[系统盘](#系统盘)** - 存放系统和代码的磁盘
- **[数据盘](#数据盘)** - 存放数据集和模型的磁盘
- **[起源云盘](#起源云盘)** - 对象存储，跨实例共享
- **[区域存储](#区域存储)** - 同区域实例间共享存储
- **[无卡创建](#无卡创建)** - 不绑定 GPU 创建实例
- **[在线升降配](#在线升降配)** - 运行中更换配置
- **[秒级计费](#秒级计费)** - 按秒收费
- **[增量镜像](#增量镜像)** - 只保存变化部分

## 实例

**实例**是一台租用的 GPU 服务器。

可以理解为：
- 远程的计算机，配有 GPU 加速卡
- 通过网络连接使用

实例支持的操作：
- 启动、关机、重启
- 在线升降配（运行中更换 GPU）
- 保存镜像
- 释放（彻底删除）

实例的状态：
- **开机中**：正在启动容器，不计费
- **运行中**：可以连接使用，正在计费
- **关机**：暂停使用，不计费，数据保留
- **已释放**：彻底删除，系统盘和数据盘清空

---

## 镜像

**镜像**是系统环境的快照，包含操作系统、深度学习框架、已安装的软件包。

镜像的作用：
1. **快速创建环境**  
   选择 PyTorch 镜像，新实例自动配置好 PyTorch 环境

2. **保存工作成果**  
   配置好环境后，保存成镜像，下次直接使用

3. **团队协作**  
   镜像可以分享给其他用户

### 镜像类型

- **官方镜像**：起源云提供的基础镜像
- **我的镜像**：你保存的镜像
- **共享镜像**：其他用户分享的镜像

<a id="增量镜像"></a>
### 增量保存 ⭐

起源云的镜像采用增量机制：
- 只保存相对基础镜像的变化部分
- 保存速度快，占用空间小
- 可以在运行中直接保存，无需关机 ⭐

---

## 存储系统

起源云提供多种存储方式，适应不同场景。

<a id="系统盘"></a>
### 系统盘（`/`）

- **路径**：根目录 `/`
- **用途**：存放系统、软件、代码
- **容量**：30-50GB
- **特点**：随镜像保存，关机后数据保留

<a id="数据盘"></a>
### 数据盘（`/root/origin-data`）

- **路径**：`/root/origin-data`
- **用途**：存放数据集、模型权重、训练日志
- **容量**：可扩展到 TB 级别
- **特点**：
  - 支持在线扩容（运行中直接扩容）⭐
  - 关机后数据保留
  - 不随镜像保存

<a id="起源云盘"></a>
### 起源云盘（`/root/origin-cloud`）⭐

**起源云盘**是对象存储服务，类似网盘。

- **路径**：`/root/origin-cloud`
- **特点**：
  - 自动挂载到所有实例
  - 跨实例共享数据
  - 实例释放后数据永久保留
  - 网页端可直接上传、下载、管理文件

**适用场景**：
- 存储常用数据集（多实例共用）
- 保存训练结果（实例释放后仍可下载）
- 团队协作共享文件

<a id="区域存储"></a>
### 区域存储（`/root/origin-share`）

**区域存储**是同一大区内实例间的共享存储空间。

- **路径**：`/root/origin-share`
- **容量**：默认 20GB 免费，上限 200GB
- **用途**：同区域实例间共享数据
- **特点**：
  - 访问速度快，本地化存储
  - 只有同一大区的实例能看到和访问
  - 跨容器共享，不受关机和镜像操作影响
  - 扩容需联系管理员

**适用场景**：
- 同区域多个实例共享数据集
- 需要高速访问的共享数据
- 团队协作（限同区域）

**与起源云盘的区别**：
- 区域存储：仅同区域可见，速度更快
- 起源云盘：全区域可见，跨区域共享

### 公共数据（`/root/origin-public`）

- **路径**：`/root/origin-public`
- **用途**：起源云提供的公开数据集
- **特点**：常用数据集预先准备好，直接使用

## 连接方式

起源云支持三种连接方式：

### JupyterLab

- **类型**：网页端
- **特点**：开箱即用，实例开机后自动启动
- **适合**：Python 开发、数据分析、快速调试

### SSH

- **类型**：命令行
- **特点**：灵活强大，支持文件传输
- **适合**：服务器管理、批量操作、自动化脚本


### VNC

- **类型**：图形桌面
- **特点**：完整的 Linux 桌面环境
- **适合**：需要图形界面的应用

---

<a id="无卡创建"></a>
## 无卡创建 ⭐

**无卡创建**允许在不绑定 GPU 的情况下先创建实例。

使用场景：
- 上传大型数据集（避免 GPU 空转浪费钱）
- 安装复杂环境（调试时不需要 GPU）
- GPU 资源紧张时提前准备环境

无卡状态下的配置由用户自行选择，实例会以较低价格计费。准备就绪后，再添加 GPU 开始训练。

---

<a id="在线升降配"></a>
## 在线升降配 ⭐

**在线升降配**允许在运行中更换 GPU 或调整配置。

操作流程：
1. 点击"修改配置"
2. 选择新的 GPU 型号或数量
3. 系统自动重启
4. 重启后新配置生效

注意事项：
- 内存中的数据会丢失（类似重启）
- 训练任务需要从 checkpoint 恢复

典型场景：
- 开发调试阶段用小卡，训练时升级到大卡
- 训练完成后降级到小卡做推理

---

<a id="秒级计费"></a>
## 计费单位

起源云采用**秒级计费**。

计费规则：
- 实例运行时按秒计费
- 关机后停止计费
- 最小计费单位：1 秒

省钱建议：
- 不用时及时关机
- 使用无卡创建准备环境
- 设置定时关机防止忘记

---

## 实例生命周期

```
创建 → 运行中 → 关机 → 释放
  ↓      ↓       ↓      ↓
 分配   计费    不计费  删除
```

### 创建

选择 GPU、镜像、配置，系统自动分配机器并开机。

### 运行中

- 可以连接使用
- 按秒计费
- 可以在线升降配、保存镜像

### 关机

- 停止计费
- 系统盘和数据盘数据保留
- 连续关机 15 天后自动释放

### 释放

- 彻底删除实例
- 系统盘、数据盘清空
- 起源云盘数据不受影响

特殊功能：
- ✅ 可以在运行中直接保存镜像（无需关机）
- ✅ 可以在运行中直接释放实例（立即停止计费）
